원사 포장 자동화 창고관리시스템(WMS) 개발

  • 개발기간 : 2021. 10 ~ 2022. 10
  • 역할 및 기여도 : 풀스택 개발자 (40%)
기여도 40%

주요업무 및 업무성과

  • 초기 프로젝트 구조 설정
  • ERWin을 이용하여 데이터모델링 된 ERD에서 MS-SQL의 DB Table 생성스크립트 추출
  • 생성한 DB Table 기반으로 Spring MVC 구조인 vo, dao, sql 파일을 생성
  • 설계된 서비스들을 토대로 기본 페이지들을 생성하여 초기 프로젝트 설정
  • 관리자 화면 구현
  • ECS와 연동으로 수동 입고 및 출고, 장비관리 기능 등 구현
  • 대시보드에서 SVG 이미지들을 추가하고 제어하여 실시간 작업현황 및 장비 모니터링 구현
  • 현장 시스템 구축, 운영 테스트와 모니터링과 이슈 대응
  • 2022년 6월부터 9월까지 현장(경북)에 방문한 일수 68일
  • 현장 내에 신규 설치된 서버에 Windows 서버 설정, WMS/ECS 설치
  • 관리자 및 작업자용 PC 8대 Windows, WMS 설정
  • 현장에 상주하며 WMS 운영 테스트, 24시간 모니터링과 이슈사항 대응
  • 현장 관리자/작업자들에게 시스템 사용 매뉴얼을 지속적으로 교육하여 사용자에 의한 이슈를 감소

개선작업 및 성과

  • WMS Service Down 지속 발생 현상
  • RAMMap 프로그램으로 서버에 메모리 누수를 유발하는 서비스를 찾아 중단 처리
  • 서버의 메모리 점유율이 기존 90%대에서 30%대를 유지
  • WMS 브라우저 화면의 로딩 속도 저하 현상
  • 브라우저 관리자 도구에서 메모리 모니터링으로 메모리 누수 유발하는 소스 개선
  • WMS에서 실시간 모니터링을 위해 지속적으로 호출되는 쿼리들의 로직 개선
  • 대시보드의 수백 개의 svg 이미지들로 창고 전체 구성된 영역에 6개 구획으로 분리 작업
  • PC에서 비정상적으로 높았던 네트워크 처리량과 GPU 사용량이 안정적으로 유지되고, 브라우저 로딩되는 시간이 기존에 10초~30초에서 1초~5초로 개선

결과

  • 기존에 설계한 WMS 모든 기능 100% 개발 완료
  • 모니터링 중 WMS 관련 발생 이슈사항 100% 처리 완료 후 현장 철수
